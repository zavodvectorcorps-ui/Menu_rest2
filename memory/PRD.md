# PRD: Личный кабинет ресторана

## Дата создания: 2026-01-26
## Последнее обновление: 2026-03-10

## Оригинальное ТЗ
Веб-приложение "Личный кабинет ресторана" по типу LunchPad. Админка + клиентское меню по QR-коду.

## Реализовано

### Мультитенантность и авторизация
- JWT (фиксированный SECRET), роли суперадмин/менеджер, управление пользователями

### Telegram-бот
- Раздел в админке, webhook, уведомления при вызовах и заказах

### Импорт меню
- .data (LunchPad) и .json форматы с импортом баннеров (type=2)
- Выбор режима: полное обновление / дополнение
- Скачивание внешних фотографий на сервер (фоновая задача с параллельной загрузкой)

### Ярлыки для блюд
- CRUD, палитра цветов, назначение на позиции, отображение в клиентском меню

### Управление меню, аналитика, настройки, столы, QR-коды
### Логотип (вилка/нож), футер "Made by Knyazev" + MK

### Предзаказы (реализовано 2026-03-10, протестировано)
- Специальный стол "Предзаказ" (is_preorder=true) создан для каждого ресторана
- Форма предзаказа в клиентском меню (имя, телефон, дата, время)
- Отдельная вкладка "Предзаказы" на странице заказов с деталями клиента
- Telegram-уведомления для предзаказов

### Массовое завершение заказов/вызовов (реализовано 2026-03-10, протестировано)
- Кнопка "Завершить все" на каждой вкладке (заказы, предзаказы, вызовы)
- Диалог подтверждения перед массовым завершением
- API: POST /api/restaurants/{id}/orders/complete-all
- API: POST /api/restaurants/{id}/staff-calls/complete-all

## Исправленные баги
- 401 Unauthorized при редактировании (JWT_SECRET теперь в .env)
- Фото не отображались (/uploads/ → /api/uploads/ для Kubernetes)
- Баннеры не импортировались из .data файла
- Дублирование столов "Предзаказ" (удалены лишние)

## Технический стек
- Frontend: React, Tailwind CSS, Shadcn/UI
- Backend: FastAPI, Motor, JWT, httpx
- Database: MongoDB
