# PRD: Личный кабинет ресторана

## Дата создания: 2026-01-26
## Последнее обновление: 2026-01-26

## Оригинальное ТЗ
Веб-приложение "Личный кабинет ресторана" по типу LunchPad для партнёров. Админка для владельца/менеджера + клиентское меню по QR-коду.

## Реализовано ✅

### Мультитенантность и авторизация
- **JWT-авторизация** с логином/паролем
- **Суперадмин** (admin/220066) с полным доступом
- **Роли**: Суперадмин (все рестораны), Менеджер (выбранные рестораны)
- **Переключатель ресторанов** в сайдбаре админки
- **Страница управления пользователями** (только для суперадмина)
- **2 тестовых ресторана**: "Мята Спортивная Тест", "Мята Центральная"

### Аналитика
- **Страница аналитики** с графиками
- **Метрики**: просмотры меню, заказы, выручка, вызовы персонала
- **Периоды**: 7/30/90 дней
- **Популярные блюда** (топ-10)
- **Конверсия** и средний чек

### Бэкенд (FastAPI + MongoDB)
- REST API с 40+ эндпоинтами
- Все данные привязаны к restaurant_id
- **Загрузка изображений**: POST /api/upload
- **Генерация QR-кодов**: GET /api/restaurants/{id}/tables/{id}/qr
- Разделы меню: "Еда", "Напитки", "Кальяны"
- Два режима отображения: карточка/компактный список
- Валюта: BYN

### Админка (React)
- **Авторизация** с формой логина
- **Переключатель ресторанов** в сайдбаре
- **Профиль**: статистика, информация о ресторане
- **Меню**: CRUD категорий и позиций, drag-and-drop, загрузка изображений
- **Заказы**: таблица заказов и вызовов персонала
- **Аналитика**: графики, метрики, популярные блюда
- **Настройки**: функции, разделы меню, типы вызовов, столы с QR-кодами
- **Пользователи**: управление доступом (суперадмин)

### Клиентское меню
- Доступ по /menu/:tableCode (без авторизации)
- Каждый ресторан имеет своё меню
- Разделы "Еда", "Напитки", "Кальяны"
- Два режима отображения позиций
- Модальное окно вызова персонала

## Технический стек
- **Frontend**: React, Tailwind CSS, Shadcn/UI, dnd-kit
- **Backend**: FastAPI, Motor, JWT, qrcode, Pillow
- **Database**: MongoDB

## API Endpoints (ключевые)
- `/api/auth/login` - авторизация
- `/api/auth/me` - текущий пользователь
- `/api/users` - CRUD пользователей (superadmin)
- `/api/restaurants` - CRUD ресторанов
- `/api/restaurants/{id}/...` - все данные по ресторану
- `/api/restaurants/{id}/analytics` - аналитика
- `/api/public/menu/{table_code}` - публичное меню
