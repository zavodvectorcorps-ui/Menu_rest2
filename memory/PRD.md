# PRD: Личный кабинет ресторана

## Дата создания: 2026-01-26
## Последнее обновление: 2026-01-26

## Оригинальное ТЗ
Веб-приложение "Личный кабинет ресторана" по типу LunchPad для партнёров. Админка для владельца/менеджера + клиентское меню по QR-коду.

## Пользовательские персоны
1. **Владелец/менеджер ресторана** - управляет меню, заказами, настройками
2. **Гость ресторана** - просматривает меню, делает заказы, вызывает персонал

## Основные требования
- Без авторизации для гостей - уникальные коды для столов
- Только русский язык
- Мятно-коричневая цветовая схема (#5DA9A4, #8D6E63)
- Переключение темы из админки
- Тестовые данные "Мята Спортивная"
- Валюта: BYN (белорусский рубль)

## Реализовано ✅

### Бэкенд (FastAPI + MongoDB)
- REST API с 25+ эндпоинтами
- Модели: Restaurant, MenuSection, Category, MenuItem, Table, Order, StaffCall, CallType, Employee, Settings
- **Разделы меню** (MenuSection): "Еда", "Напитки", "Кальяны"
- **Типы вызовов** (CallType): "Вызов официанта", "Вызов кальянного мастера", "Попросить счёт"
- Рекламные баннеры (is_banner в MenuItem) без цены
- Настройки Telegram-бота (telegram_bot_token, telegram_chat_id)
- Статистика: просмотры меню, вызовы персонала, заказы
- Автоматическое заполнение тестовыми данными (seed)

### Админка (React)
- **Профиль**: виджеты статистики, информация о ресторане, промо-блок редактора
- **Меню**: 
  - CRUD категорий с привязкой к разделам меню
  - CRUD позиций с бейджами (хит, новинка, острое, акция)
  - Создание рекламных баннеров (без цены)
  - Drag-and-drop сортировка
- **Заказы**: таблица заказов, вкладка вызовов персонала, обновление статусов
- **Настройки**: 
  - Функции (переключатели модулей)
  - О ресторане (название, адрес, контакты, режим работы)
  - **Разделы меню** (CRUD)
  - **Типы вызовов** (CRUD с шаблонами сообщений Telegram)
  - Столы (CRUD, уникальные коды, QR-ссылки)
  - Сотрудники (CRUD с Telegram ID)
  - **Интеграции** (Telegram Bot Token, Chat ID)
  - Оформление (тема, цвета)
- **Справочный центр**: FAQ с поиском и категориями
- **Поддержка**: форма обращения

### Клиентское меню
- Доступ по /menu/:tableCode
- Мобильная адаптивность
- **Табы разделов**: "Еда", "Напитки", "Кальяны"
- Категории блюд внутри выбранного раздела
- Карточки блюд с бейджами
- **Рекламные баннеры** (картинки на всю ширину)
- Добавление в корзину (с валютой BYN)
- Оформление заказа с комментарием
- **Модальное окно вызова** с настраиваемыми типами
- Контактная информация ресторана

## Технический стек
- **Frontend**: React, Tailwind CSS, Shadcn/UI, Lucide Icons, dnd-kit
- **Backend**: FastAPI, Motor (async MongoDB)
- **Database**: MongoDB
- **Fonts**: Manrope (headings), Inter (body)

## API Endpoints
- `/api/restaurant` - информация о ресторане
- `/api/menu-sections` - CRUD разделов меню
- `/api/categories` - CRUD категорий (с section_id)
- `/api/menu-items` - CRUD позиций (с is_banner)
- `/api/call-types` - CRUD типов вызовов
- `/api/tables` - CRUD столов
- `/api/orders` - CRUD заказов
- `/api/staff-calls` - вызовы персонала
- `/api/settings` - настройки (включая Telegram)
- `/api/public/menu/{table_code}` - публичное меню для гостей
